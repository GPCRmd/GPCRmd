About the heatmap:

This plot compares residue interaction frequency among several GPCR dynamic simulations. Interaction frequencies have been calculated with GetContacts scripts, created by Rasmus Fonseca (fonseca.rasmus@gmail.com) and Anthony Ma (anthonyma27@gmail.com). Frequency values are obtained from the proportion of frames in which two residues are considered to be interacting, according to GetContacts interaction criteria. From these frequency values, a dendrogram clustering is computed and displayed using plotly python library.

Protein residues are labeled using generic GPCR residue numbers (GPCRdb structure-based numbering). This numbering system differs for each GPCR class, and so a multi-class index has been used on the heatmap's axis. Only residue pairs with a minimum of 50% interaction frequency in at least one simulation are displayed. Same-helix interactions are not displayed either. Orthosteric ligands of complex forms have been collectively labeled as "Lig", and are treated as a single residue (even if they have more than one, as peptidic ligands have).

How to use this heatmap:

  1. Select the simulation datasets to display
  2. Select the type of interaction which frequencies display. Total interaction frequency reffers to the percentage of frames on which the residue pair is interacting in any of the interaction criteria
  3. Select the partners of the interactions to be displayed (ligand-protein, protein-protein or both)
  4. Select the number of clusters to display in the dendrogram
  5. Check "Show reversed residue pairs" to display interactions for the reversed residue pair. (Eg: 5x43-7x32 and 7x32-5x43)
  6. Click apply to load the new interactions selected.

More information of a specific interaction can be displayed by clicking uppon its cell.

Customized heatmaps

If you wish to analyze a specific set of simulations, use the "Select simulations" dropdown and then press "apply" . The whole page will reload and a new set of heatmaps comprising exclusively the selected simulations will be displayed. Beware that a large selection may result in a long loading time.

About the TopX interactions flareplots:

On these two plots are represented the top most frequent interactions (on average) in each of the clusters. Each line represents one of these top interactions, and its color is based in the cluster average frequency of this interaction. The color scale is the same as the heatmap.

How to use these flareplots:

  1. To select a new cluster to display, use the corresponding dropup under each flareplot. Cluster 1 and Cluster 2 are shown by default.
  2. To choose the numbering scheme applied in the flareplot interactions, use the corresponding dropup. Both flareplots use class A numbering (Ballesteros-Weinstein) by default.
  3. To select a position on the flareplot, click on it. If avalible, this position will be displayed on the structure visualizer below.
  4. Press "show in structure" to disable or reenable the visualization of selected positions in the viewers bellow.

All flareplots were created using FlarePlots, by Rasmus Fonseca (fonseca.rasmus@gmail.com) and Anthony Ma (anthonyma27@gmail.com).

About the cluster simulation viewers:

Use these viewers to visualize receptors from the clusters selected in the "TopX intearctions flareplots" above.

How to use these viewers:

  1. Use the top-right dropdown to select the simulation to display. Notice that only simulations of the selected cluster are avalible.
  2. Use the top-left dropdown to select the trajectory file of this simulation to display.
  3. Press play to reproduce the simulation on the viewer.
  4. Click on any residue in the flareplot to display it on the corresponding viewer.
  5. Click on any interaction in the flareplot to display its corresonding residues in the viewer. If one of the residues is a peptidic ligand, only the ligand residues closer than 3 amstrongs to the non-ligand residue will be displayed.
  6. For a more detailed visualization of this simulation, click on "open with GPCRmd viewer".

These visualizers run on NGL viewer.
